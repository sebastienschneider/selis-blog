Every developer has their toolkit. Here's mine — not aspirational picks, but the things I actually use day to day.

<!--more-->

## Editor

**VS Code** remains my primary editor. It's fast enough, extensible, and has excellent Git integration. The key extensions I can't live without:

- **GitLens** — makes Git history actually useful
- **Prettier** — consistent formatting without thinking about it
- **Error Lens** — inline error highlighting

## Terminal

A good terminal setup saves hours over time. I use:

- **zsh** with a minimal prompt — no fancy themes, just the info I need
- **tmux** for session management when working on remote machines
- **fzf** for fuzzy finding files and command history

## Version Control

**Git** and **GitHub** for everything. A few habits that help:

- Small, focused commits with clear messages
- Feature branches for anything non-trivial
- Pull requests even for solo projects — they force you to review your own code

## Deployment

**Netlify** for static sites like this blog. The deploy-on-push workflow is seamless — push to main, and the site updates in seconds. The free tier is generous enough for most personal projects.

## Writing

Blog posts are written in **Markdown** using VS Code. No special writing apps needed. The preview pane works well enough, and keeping posts in the same editor as code means less context-switching.

## What I Don't Use

Just as important as what you use is what you skip:

- No Docker for local development (unless the project demands it)
- No complex build pipelines for personal projects
- No productivity apps — a plain text file beats most task managers

The best toolkit is the one you actually know well. Depth beats breadth.
